The Code Behind It All: HTML and CSS

Computers, like humans, use languages as instructions on what to do when it reads some lines of codes. Lines of code tell the browser how to put together the parts of the website (text, images, animations, etc.) so that the website is structure in a specific way. They also tell the browser what content to put on the website so every user can see the same information. In this project, we used two types of coding language: HTML and CSS. We used HTML as a basic framework of our website. It allowed us to put in the content of the website but not stylize it. On the other hand, CSS is the coding language that changes the format of the website to alter the appearance. This is how coders can take the same website and truely make it their own; it is what allows for such a diverse visual range of websites on the internet. This webpage is an explination of these two languages and how they come together to shape websites such as this one.

Hypertext Markup Language

Background: HTML stands for Hypertext Markup Language. This means that it is a language that links to other texts together (hypertext) and tells the computer how to assemble the webpage's content (markup). This language was created by computer scientist Tim Berners-Lee in 1989 as the standard language of his new invention, the World Wide Web. He thought of HTML while working at CERN, the European Laboratory for Particle Physics, as a way for researchers to share their research and work together on the Web. His idea was to link research papers together so scientists could cross-reference multiple documents at a time, which was the start of hypertext for the Web. Eventually, HTML became the main language for programing websites on the Web and it still is to this day.

Tim Berners-Lee, Inventor of the World Wide Web and of HTML

How it works: HTML works through a series of tags, which are labels that tells the browser what content can be classified as. They are placed in angled brackets (<>). These brackets tell the browser that what is inside them is ment for the browser to read, not display on the webpage. For example, a paragraph can be defined in HTML as a p. Some tags consist of a start tage and an end tag. The start tag is placed at the start of its content and the end tag closes the content. The difference between them is that there is a forward slash (/) inside the brackets before the tag for the end tag that isn't present for the start tag. When refering to the start and end tags along with its content in between, it is called an element. Browsers read the elements as a whole to understand what type of content it is and what it is. In addition, tags can be modified with attributes. Attributes are placed inside the start tag to add additional information for the browser to read about that tag's element. An example would be width for the image tag to change the dimensions of the image. Another attribute example is align for tables that changes whether a table is aligned to the left, venter, or right. HTML document code is seperated by the head tag and the body tag. The head tag is where information about the webpage and CSS styling code is placed. The body is where all of the visible content of the webpage is place so it can be seen on the actual page. Overall, tags and elements are the basis of HTML, telling the browser what to show on the webpage and being the structure of it.

Cascading Style Sheets

Background: The other language used to create my website is CSS, which stands for Cascading Style Sheets. This language doesn't change the content of a webpage but its visual appearance. It has the ability to move elements around, change colors, and other properties (style sheet). "Cascading" is how CSS works by arranging parts of the webpage in different ways depending on the code. The idea of CSS was proposed by Hakon Wium Lie, the Chief Technology Officer of Opera, a browser. He thought of the idea while working with Tim Berners-Lee at CERN. The goal of Lie's idea was to create a syle sheet that allowed web designers and developers to have more control when styling their websites. In 1996, CSS 1 was released as the first version of the new style sheets for the World Wide Web. CSS is still used to customize the look of websites, creating unique looking websites with the same HTML code.

Hakon Wium Lie, the man who proposed CSS

How it works: CSS is coded in a similar manner to HTML but there are some differences. CSS code is placed inside the style tags inside the head tags at the top of the page. This tells the browser to read the code inside the style tags in CSS instead of HTML. When it comes to writing the actual code, there are more differences. The first part of a line of code is the selector, which is just the name of the elements the coder is choosing to stylize. Selectors can be the general names, ids, or classes. The general name would be like p for paragraph. An id is a name that can be given to a specific element in the HTML body so the coder can refer to that element as a selector. It is written with a pound sign (#) and the name of the id. For example, an id with the name "intro_paragraph" would be #intro_paragraph in CSS. Finally, a class allows for multiple elements that you want to share the same CSS manipulation to be grouped together. It has a similar CSS selector but the pound sign is replaced with a period (.). An example would be a class of "computer_parts" would be written .computer-parts. After the selector comes a set of braces/ curly brackets ({ }). Inside these brackets is the CSS property (size, color, width, etc) followed by a colon (:). Then there is the value of that property ended with a semicolon (;).
